# Part 1. 웹 포털 상세 설계

바르질라 Wärtsilä Online과 콩스버그 K-IMS를 참고하여, 우리 회사 웹 포털에 들어가야 할 기능을 최대한 상세하게 정리합니다.

---

## 1. 로그인/대시보드

### 1-1. 로그인 및 계정 관리

| 기능 | 상세 | 참고 |
|------|------|------|
| **고객사 계정** | 선주/관리사별 회사 계정 생성. 담당자 여러 명 등록 가능 | 바르질라 Online: 회사 단위 계정 |
| **권한 관리** | 관리자, 일반 사용자, 열람 전용 등 권한 구분 | 콩스버그 K-IMS: 역할 기반 접근 |
| **SSO 연동** | 고객사 기존 시스템과 싱글사인온 가능 (대형 고객) | 바르질라: ERP PunchOut 연동 |
| **다국어** | 한국어, 영어, 중국어 등 (선주 국적에 따라) | — |

### 1-2. 메인 대시보드 (로그인 후 첫 화면)

| 요소 | 상세 |
|------|------|
| **내 선박 목록** | 고객사 보유 선박 리스트 + 각 선박 상태 아이콘 (정상/주의/점검 필요) |
| **진행 중인 서비스** | 현재 진행 중인 서비스 요청 건 목록 + 단계 표시 |
| **최근 알림** | 점검 리마인더, 기술 회보, 서비스 상태 변경 등 최신 알림 |
| **빠른 메뉴** | 서비스 요청, 문서 검색, 담당자 연락 바로가기 |
| **공지사항** | HD현대마린솔루션테크 공지, 기술 업데이트 등 |

| 영역 | 내용 |
|------|------|

## 3. 서비스 요청 및 추적 (Service Request)

### 3-1. 서비스 요청 작성
> 바르질라 TechRequest + 콩스버그 Support/Spare Part Request 참고

| 단계 | 내용 |
|------|------|
| **① 선박 선택** | 드롭다운에서 선박 선택 (또는 검색) |
| **② 요청 유형 선택** | 기술 지원 / 헬스체크 요청 / 수리 요청 / 긴급 수리 / 기타 |
| **③ 대상 설비 선택** | 해당 선박의 배전반/기자재 목록에서 선택 |
| **④ 상세 내용 작성** | 증상, 요청 사항 텍스트 입력 |
| **⑤ 파일 첨부** | 사진, 동영상, 도면, 기존 리포트 등 첨부 |
| **⑥ 희망 일정** | 입거 예정일, 희망 작업 시기 입력 |
| **⑦ 긴급도 선택** | 일반 / 우선 / 긴급 |
| **⑧ 제출** | → 접수 확인 이메일 + 웹 알림 자동 발송 |

### 3-2. 서비스 추적 대시보드

| 기능 | 상세 |
|------|------|
| **진행 단계 표시** | 접수 → PM 배정 → 견적 발송 → 고객 승인 → 공사 진행 → 완료 (스텝 바 UI) |
| **단계별 일자** | 각 단계 진입 일자 + 경과일 자동 표시 |
| **담당자 정보** | 현재 담당 PM 이름, 연락처, 웹 메시지 버튼 |
| **첨부 문서** | 견적서, 작업 계획서, 완료 보고서 등 단계별 문서 |
| **코멘트/소통** | 고객↔PM 간 웹 내 메시지 교환 (이메일 알림 연동) |
| **이력 로그** | 모든 상태 변경, 메시지, 문서 업로드의 타임스탬프 기록 |

### 3-3. 서비스 이력 조회

| 기능 | 상세 |
|------|------|
| **전체 이력 검색** | 선박별, 기간별, 유형별, 키워드 검색 |
| **이력 상세 보기** | 과거 완료 건의 전체 진행 과정, 첨부 문서, 코멘트 열람 |
| **통계** | 선박별 서비스 요청 건수, 유형 분포, 평균 처리 시간 등 |

## 4. 점검/수리 이력 (Service History)

### 4-1. 헬스체크 이력

| 항목 | 내용 |
|------|------|
| **방문 일자** | 헬스체크 수행일 |
| **점검 엔지니어** | 담당 엔지니어 이름 |
| **점검 항목** | 절연저항, 차단기 동작시간, 온도, 외관 검사 등 각 항목별 결과 |
| **측정값** | 각 항목별 실제 측정값 (숫자) |
| **판정** | 정상 / 주의 / 위험 (항목별) |
| **리포트** | 해당 회차 점검 리포트 열람/다운로드 |
| **권고사항** | 엔지니어 권고 내용 |
| **사진** | 점검 시 촬영 사진 |

### 4-2. 수리 이력

| 항목 | 내용 |
|------|------|
| **작업 건명** | "MSB #1 ACB 교체" 등 |
| **작업 일자** | 수리 시작/완료일 |
| **작업 내용** | 상세 작업 내역 |
| **교체 부품** | 교체된 부품명, 모델, 수량 |
| **작업 인원** | 투입 인원 수, 작업일 수 |
| **비용** | 총 비용 (선택적 공개 여부 설정 가능) |
| **완료 리포트** | 작업 완료 리포트 열람/다운로드 |
| **보증 기간** | 해당 수리 건 보증 종료일 |

### 4-3. 타임라인 뷰
> 콩스버그 K-IMS: 선박별 이벤트 시간순 표시

| 기능 | 상세 |
|------|------|
| **시간순 표시** | 해당 선박의 전체 서비스 히스토리를 타임라인으로 시각화 |
| **이벤트 유형 필터** | 헬스체크, 수리, 서비스 요청, 부품 교체 등 유형별 필터 |
| **클릭 → 상세** | 타임라인 항목 클릭 → 해당 건 상세 페이지로 이동 |

## 5. 리포트 시각화 대시보드 (Report Dashboard)

### 5-1. 선박별 건강도 대시보드
> 콩스버그 K-IMS: 대시보드, 알람 통계, 트렌드 & 리포트

| 요소 | 상세 |
|------|------|
| **종합 건강도 점수** | 100점 만점 게이지 차트 + 전월 대비 변화 |
| **배전반별 상태 카드** | MSB #1, MSB #2, ESB 등 각각의 점수 + 정상/주의/위험 |
| **주의 항목 하이라이트** | 주의/위험 판정 항목 상단 강조 표시 |

### 5-2. 측정값 추이 그래프

| 측정 항목 | 그래프 유형 | 표시 내용 |
|----------|-----------|----------|
| **절연저항** | 꺾은선 그래프 (시계열) | 최근 12~24개월 추이 + 정상/주의/위험 기준선 |
| **차단기 동작시간** | 가로 막대 차트 | 정상 범위 vs 현재값 비교 (색상 구분) |
| **온도** | 가로 막대 or 히트맵 | 부위별(R/S/T상) 온도 + 기준치 초과 강조 |
| **전류/전압** | 꺾은선 그래프 | 불평형률 추이, 과부하 이벤트 표시 |

### 5-3. 부품 수명 관리

| 요소 | 상세 |
|------|------|
| **부품별 잔여 수명** | 프로그레스 바 (남은 수명 %) + 예상 교체 시기 |
| **교체 이력** | 해당 부품의 과거 교체 일자, 사용 기간 |
| **교체 권장 알림** | 잔여 수명이 임계값 이하 → 자동 알림 |

### 5-4. AI 분석 코멘트 (향후)

| 요소 | 상세 |
|------|------|
| **트렌드 분석** | "절연저항이 월평균 27MΩ씩 하락 중. 6개월 후 위험 진입 예상" |
| **비교 분석** | "동일 타입 선박 10척 중 하위 20%에 해당" |
| **권고사항** | "다음 입거 시 정밀 점검 권장" |
| **향후 전망 그래프** | 현재 추세 기반 예측 트렌드라인 (점선) |

### 5-5. 리포트 생성/출력

| 기능 | 상세 |
|------|------|
| **자동 리포트 생성** | 대시보드 데이터 기반 PDF 리포트 자동 생성 |
| **수동 커스텀** | PM이 코멘트 추가, 항목 선택 후 커스텀 리포트 생성 |
| **리포트 이력** | 과거 발행된 리포트 전체 목록 + 열람/다운로드 |
| **이메일 발송** | 생성된 리포트를 웹에서 바로 고객에게 이메일 발송 |

## 7. 알림/리마인더 시스템 (Notification)

### 7-1. 알림 종류

| 알림 유형 | 트리거 조건 | 수신 대상 |
|----------|-----------|----------|
| **정기 점검 알림** | 마지막 헬스체크 후 N개월 경과 | 고객 + PM |
| **부품 교체 알림** | 부품 설치 후 N년 경과 or 개폐 횟수 초과 | 고객 + PM |
| **입거 사전 알림** | 등록된 입거 예정일 D-30, D-14, D-7 | 고객 + PM |
| **서비스 진행 알림** | 견적 발송, PO 수신, 공사 시작, 공사 완료 등 | 고객 |
| **기술 회보 알림** | 해당 고객 선박에 적용되는 새 회보 발행 | 고객 |
| **리포트 발행 알림** | 새 점검 리포트 업로드 시 | 고객 |
| **장기 미응답 알림** | 견적 발송 후 N일간 고객 응답 없을 시 | PM (내부) |

### 7-2. 알림 전달 방식

| 방식 | 상세 |
|------|------|
| **웹 내 알림** | 대시보드 상단 벨 아이콘 + 알림 목록 |
| **이메일** | 알림 내용을 이메일로 동시 발송 |
| **알림 설정** | 고객이 알림 유형별로 ON/OFF 설정 가능 |

# Part 2. 웹 포털 외 — 선도기업 격차 해소 추가 방안

웹 포털 구축 외에, 선도기업과의 격차를 줄이기 위한 다른 방안들입니다.

---

## 방안 1. 라이프사이클 계약 (Lifecycle Agreement) 도입

> 바르질라 핵심 전략: 일회성 AS → 장기 라이프사이클 계약으로 전환

### 현재
- 고장 → 수리 → 완료 (건별 일회성)
- 매출이 불규칙 (고장이 나야 수익)

### 제안
- **배전반 라이프사이클 계약** 판매: "연간 정기 점검 + 긴급 대응 + 기술 지원"을 묶어서 **연간 계약**
- 바르질라는 아일랜드 해군 4척과 5년 계약, LNG 운반선 14척과 장기 계약 등 적극 추진 중
- 우리도 주요 선주 대상으로 **"배전반 종합 관리 계약"** 제안 가능

### 효과
- 일회성 매출 → **반복 매출(recurring revenue)**
- 선주 입장: 유지보수 비용 예측 가능 + 긴급 대응 보장
- 우리 입장: 안정적 매출 + 선주 이탈 방지 (장기 계약 = lock-in)

---

## 방안 2. 원격 기술 지원 (Remote Expert Support)

> 바르질라 Smart Support Centre + Expert Insight 참고

### 현재
- 기술 문의 → 전화/메일 → 엔지니어 직접 방문

### 제안
- **화상/AR 원격 지원**: 선상 기관사가 스마트폰/태블릿으로 배전반을 촬영 → 우리 엔지니어가 원격으로 보면서 실시간 가이드
- 바르질라는 Smart Support Centre에서 원격 모니터링 + 전문가 지원 → "거의 모든 이슈를 원격으로 해결" 주장
- 웹 포털에 **화상 통화 / 화면 공유 기능** 내장

### 효과
- 엔지니어 출장 횟수 감소 → 비용 절감
- 긴급 상황 시 **즉시 대응** 가능 (출장 대기 없이)
- 글로벌 고객 대응 범위 확대

---

## 방안 3. 엔지니어 역량 표준화 + AI 어시스턴트

### 현재
- 엔지니어 역량이 사람마다 다름
- 베테랑 퇴직 시 노하우 유실
- 리포트 품질 편차 큼

### 제안
- **헬스체크 표준 프로세스 디지털화**: 체크리스트 앱 → 엔지니어가 단계별로 따라감 → 측정값 입력 → 자동 판정
- **AI 트러블슈팅 어시스턴트**: 현장에서 "이 증상이면 원인이 뭐야?" → AI가 과거 사례 기반 답변
  - 참고: SmartSeas AI — 엔지니어가 QR 코드 스캔 → 즉시 트러블슈팅 가이드 (오프라인에서도)
- **노하우 DB 구축**: 과거 수리 건의 원인-조치-결과를 체계적으로 축적 → AI 검색

### 효과
- 신입 엔지니어도 **표준화된 품질** 보장
- 베테랑 노하우가 **시스템에 축적** → 퇴직해도 유실 안 됨
- 리포트 품질 편차 해소

---

## 방안 4. 데이터 기반 선제적 영업 (Proactive Service)

> 바르질라: "Expert Insight"로 고객에게 선제적 조언 제공

### 현재
- 고객이 연락해야 AS 시작
- 헬스체크도 PM 판단으로 스케줄링

### 제안
- **축적된 데이터로 영업 타겟팅**: "이 선박은 배전반 설치 5년 경과 + 동일 타입 평균 교체 주기 접근 → 헬스체크 제안"
- **자동 마케팅**: 웹 포털 알림 + 이메일로 "점검 시기 도래" 자동 안내
- **선주별 맞춤 제안**: 과거 수리 패턴 분석 → "이 선주는 매년 이 시기에 입거 → 3개월 전에 접근"

### 효과
- **수동 영업 → 데이터 기반 자동 영업**
- 선주가 찾아오기 전에 우리가 먼저 → 수주 선점
- 경쟁 AS 업체 대비 차별화

---

## 방안 5. 서비스 품질 인증/브랜드화

### 현재
- "AS 했다" → 끝. 서비스 품질에 대한 객관적 지표나 인증 없음

### 제안
- **서비스 품질 지표 공개**: 평균 응답 시간, 평균 처리 시간, 고객 만족도 → 웹 포털에 게시
- **서비스 리포트 브랜딩**: "HD Marine Solution Tech Certified Health Check Report" → 표준화된 고품질 리포트 브랜드
- **서비스 이력 인증서**: 선박 매각 시 "이 선박은 정기적으로 인증된 점검을 받았습니다" 증명 → 중고선 가치 향상

### 효과
- 서비스 자체가 **브랜드** 가치를 가짐
- 선주 입장: 중고선 매각 시 유리 → 우리 서비스를 받을 인센티브
- 경쟁사 대비 **차별화된 포지셔닝**

---

## 방안 6. 교육/세미나 프로그램

> 콩스버그/바르질라: 고객 대상 기술 교육, 웨비나 제공

### 현재
- 기술 정보 전달이 리포트/메일 중심

### 제안
- **온라인 웨비나**: "배전반 유지보수 베스트 프랙티스", "신규 규정 대응" 등 정기 웨비나
- **기술 교육 영상**: 웹 포털에 교육 동영상 라이브러리 (배전반 점검 방법, 비상 절차 등)
- **고객 기관사 대상 교육**: 선상에서 기관사가 직접 할 수 있는 기본 점검 교육

### 효과
- 고객과의 **지속적 접점** 확보 (AS 건 없어도 관계 유지)
- 기술 전문성 어필 → **신뢰도 향상**
- 기관사가 이상 징후를 빨리 인식 → 조기에 AS 요청 → 수주 기회

---

## 방안 7. 파트너십/생태계 확대

### 현재
- 배전반 AS만 단독 수행

### 제안
- **타 기자재 AS 업체와 협업**: 배전반 + 발전기 + 모터 등 전기 계통 통합 서비스 패키지
- **선급/검사 기관 연계**: 선급 검사 스케줄과 연동 → "선급 검사 전 사전 점검" 서비스
- **보험사 연계**: 정기 점검 이력 → 보험료 할인 연계 (선주 인센티브)

### 효과
- **단품 AS → 토탈 솔루션** 포지셔닝
- 고객 입장: 여러 업체 따로 관리 → 우리 하나로 해결
- 추가 수익원 확보
