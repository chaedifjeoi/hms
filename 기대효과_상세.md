## 1. 매출 구조 전환 — 불규칙 건별 매출 → 안정적 반복 매출

### 현재 문제

```
고장 발생 → 연락 옴 → 수리 → 청구 → 끝 → (다음 고장까지 매출 0)
```

- 매출 예측 불가. 분기마다 들쭉날쭉
- 고객이 안 부르면 매출이 없음
- 건마다 견적 비교 → 가격 경쟁에 노출

### 웹 포털 + 라이프사이클 계약 도입 후

```
웹 포털에서 고객이 매일 접속 → 데이터 축적 → 선제적 서비스 제안
→ 라이프사이클 계약 체결 → 연간 고정 요금 → 안정적 반복 매출
```

### 매출 시뮬레이션

```
■ Tier 1 (Basic) — 선박당 연 ₩10M
  파일럿 50척 × ₩10M = ₩5억/년

■ Tier 2 (Standard) — 선박당 연 ₩20M
  확대 50척 × ₩20M = ₩10억/년

■ Tier 3 (Premium) — 선박당 연 ₩50M
  핵심 고객 20척 × ₩50M = ₩10억/년

■ 합계: 연간 ₩25억 반복 매출 (3년차 목표)

+ 계약 고객의 추가 수리 건 → 자연스러운 추가 수주 (연 ₩10억+ 추정)
= 총 ₩35억+ 연간 매출 기반 확보
```

### 바르질라 대비

- 바르질라: 서비스 매출 **€34억 (약 5조 원)**, 전체의 53%
- 바르질라: 라이프사이클 계약 갱신율 **90% 이상**
- **우리가 120척만 계약해도 연 ₩25억 반복 매출. 9,890척 네트워크의 1.2%에 불과.**
- 확대 가능성: 9,890척 중 5%(~500척)만 계약해도 → **연 ₩100억+ 반복 매출**

## 2. 고객 Lock-in — 한 번 들어오면 나가지 않는 구조

### 현재 문제

- 건별 거래 → 다음 건에서 경쟁사에 줄 수 있음
- 가격만으로 선택 → 경쟁사가 싸면 빼앗김
- 고객과의 관계가 얕음 (AS 건 없으면 접점 없음)

### 웹 포털이 만드는 Lock-in 구조

```
① 웹 포털에 고객이 접속 시작
    ↓
② 호선정보, 서비스 이력, 리포트가 웹에 축적
    ↓
③ 3년치 배전반 데이터가 우리 플랫폼에 쌓임
    ↓
④ 이 데이터는 경쟁사에 없음
    ↓
⑤ 고객이 경쟁사로 전환하면 → 3년치 이력/데이터 전부 잃음
    ↓
⑥ 전환 비용(Switching Cost)이 높아짐 → 이탈 안 함
```

### 구체적 Lock-in 요소

| Lock-in 요소 | 설명 | 전환 시 고객이 잃는 것 |
|-------------|------|----------------------|
| **서비스 이력** | 3~5년치 점검/수리 이력이 웹에 축적 | 과거 정비 기록 전부 |
| **측정 데이터** | 절연저항, 차단기 동작시간 등 추이 데이터 | 트렌드 분석 기반 |
| **리포트 아카이브** | 시각화 리포트 전체 이력 | 과거 건강도 비교 불가 |
| **부품 수명 추적** | 각 부품의 잔여 수명 데이터 | 교체 시기 예측 불가 |
| **전담 PM 관계** | 선박 사정을 아는 전담 PM | 처음부터 관계 구축 필요 |
| **알림 설정** | 맞춤형 알림 체계 | 수동 관리로 복귀 |

> **바르질라 실증: 라이프사이클 계약 갱신율 90% 이상.** 한 번 계약하면 거의 이탈하지 않는다. 이유: 축적된 데이터 + 편의성 + 전환 비용.

## 3. 수주 속도 향상 — 리포트가 영업을 한다

### 현재 문제

- 헬스체크 리포트가 숫자/텍스트 나열 → 선주가 심각성을 체감하지 못함
- "절연저항 5.2MΩ" 라고 써도 → 선주: "그래서 위험한 건가요?"
- 리포트 → 선주 검토 → 결정까지 수주~수개월 소요

### 웹 포털 시각화 리포트 도입 후

| 현재 리포트 | 시각화 리포트 |
|------------|-------------|
| "절연저항: 5.2MΩ" (숫자) | 게이지 차트: **빨간색 구간에 바늘** (한눈에 "위험") |
| "전월 대비 하락" (텍스트) | 꺾은선 그래프: **3개월 연속 하락 추이** (눈에 보임) |
| "교체 권장" (한 줄) | 부품 수명 프로그레스 바: **잔여 12%** + AI: "3개월 후 위험 진입 예상" |

### 수주 속도 변화 예상

```
■ 현재
리포트 전달 → 선주 검토 (1~2주) → 사내 결재 (1~2주) → 수리 발주 (2~3일)
= 총 3~5주

■ 시각화 리포트 후
대시보드 열람 → 위험 항목 즉시 인지 → 빠른 결재 → 수리 발주
= 총 1~2주 (50~70% 단축)
```

> **같은 데이터도 보여주는 방식이 다르면 선주의 의사결정이 달라진다.** 리포트 품질 = 수주 속도.

### 기대 효과

- 수리 건당 수주 기간: **3~5주 → 1~2주** (50~70% 단축)
- 연간 수리 수주: 건수 동일해도 **회전율 상승 → 매출 조기 인식**
- 리포트를 본 선주가 **추가 수리 건을 자발적으로 발주**하는 비율 증가

## 4. PM 업무 부하 감소 — 반복 전화 제거

### 현재 문제

PM이 받는 전화/메일의 상당수가 **단순 문의**:

| 문의 유형 | 빈도 | 소요 시간 |
|----------|------|----------|
| "우리 배 배전반 모델이 뭐예요?" | 주 5~10건 | 건당 10~20분 (내부 시스템 조회 후 회신) |
| "저번 수리 뭘 했었죠?" | 주 3~5건 | 건당 15~30분 (과거 기록 찾기) |
| "지금 진행 상황이 어떻게 돼요?" | 주 10~15건 | 건당 5~10분 |
| "도면 보내주세요" | 주 3~5건 | 건당 10~15분 (파일 찾아서 메일 발송) |
| "견적 보내주세요" | 주 5~8건 | 건당 30분~1시간 |

### 웹 포털 도입 후

| 문의 유형 | 웹 포털 대체 | PM 절감 시간 |
|----------|------------|------------|
| 배전반 모델 문의 | **고객이 웹에서 직접 검색** | 주 2~3시간 절감 |
| 과거 수리 이력 문의 | **고객이 웹에서 직접 조회** | 주 1~2시간 절감 |
| 진행 상황 문의 | **웹에서 실시간 추적** | 주 2~3시간 절감 |
| 도면/문서 요청 | **웹 라이브러리에서 24/7 셀프서비스** | 주 1~2시간 절감 |
| 견적 요청 | **웹 요청 + AI 자동 산출** | 주 3~5시간 절감 |

### 정량적 절감

```
■ PM 1인당 주간 절감: 약 10~15시간
■ PM 5명 기준 월간 절감: 200~300시간
■ 연간: 2,400~3,600시간

이 시간을 고부가가치 업무(영업, 고객 관계, 기술 자문)에 투입 가능
→ 같은 인원으로 더 많은 건을 클로징
```

> **PM이 전화 받고 파일 찾는 시간을 → 영업하고 수주하는 시간으로 전환.**

## 5. 출장 비용 절감 — 원격 지원으로 50% 절감

### 현재 출장 비용 구조

```
연간 출장 건수: ~100건 (가정)
평균 출장비: ₩3M (항공+숙박+일당, 해외 포함)
연간 총 출장비: ₩300M (₩3억)
```

### 웹 포털 + 원격 지원 도입 후

| 유형 | 건수 | 단가 | 비용 |
|------|------|------|------|
| 원격 해결 (출장 불필요) | 50건 | ₩0 | ₩0 |
| 사전 진단 후 정확한 출장 | 30건 | ₩3M | ₩90M |
| 기존 방식 출장 | 20건 | ₩3M | ₩60M |
| **합계** | 100건 | — | **₩150M** |

```
연간 절감액: ₩300M - ₩150M = ₩150M (50% 절감)
```

### 바르질라 벤치마크

- 바르질라 Smart Support Centre: **95% 원격 해결**
- 미 해군 AR 원격 지원: 수리 속도 **47% 향상**, 작업 오류 **53% → 13%**로 감소
- 우리가 보수적으로 50%만 원격 해결해도 → **연 ₩1.5억 절감**

### 대응 속도 변화

| 상황 | 현재 | 원격 지원 후 |
|------|------|------------|
| 국내 선박 | 1~3일 (출장 이동) | **수 시간** |
| 해외 선박 | 1~2주 (비자+항공+이동) | **수 시간** (거리 무관) |
| 긴급 상황 | "최대한 빨리 가겠습니다" | **1시간 이내 원격 대응** |

## 6. 선제적 영업 — 고객이 부르기 전에 우리가 먼저

### 현재 문제

- **수동적 영업**: 고객이 고장 나서 전화해야 AS 시작
- 고장이 없으면 접점이 없음
- 경쟁사가 먼저 접근하면 뺏길 수 있음

### 웹 포털 알림 시스템이 만드는 선제적 영업

```
■ 데이터 축적
웹 포털에 헬스체크 데이터가 쌓임
    ↓
■ 자동 분석
"이 선박 MSB #1 절연저항이 3개월 연속 하락 중"
"ACB 설치 후 5년 경과, 동일 타입 평균 교체 주기 접근"
    ↓
■ 선제적 알림
웹 포털 알림 + 이메일: "점검 시기가 도래했습니다"
    ↓
■ 우리가 먼저 접근
PM: "리포트를 보니 이 부분 점검이 필요합니다. 다음 입거 시 같이 보시죠"
    ↓
■ 수주 선점
경쟁사가 접근하기 전에 이미 우리가 자리 잡음
```

### 선제적 영업의 구체적 시나리오

| 시나리오 | 현재 | 웹 포털 후 |
|---------|------|----------|
| 부품 교체 시기 | 고장 나야 교체 | "교체 시기 3개월 전" 알림 → 사전 제안 → 입거 시 동시 수행 |
| 입거 일정 | 고객이 알려줘야 앎 | 웹에 입거 일정 등록 → D-30 자동 알림 → 사전 작업 계획 제안 |
| 절연 열화 | 다음 점검 때 발견 | 추이 그래프로 조기 감지 → 즉시 정밀 점검 제안 |
| 동종 선박 비교 | 불가 | "동일 타입 선박 50척 중 하위 20%" → 즉시 조치 제안 |

> **수동 영업 → 데이터 기반 자동 영업.** 선주가 찾아오기 전에 우리가 먼저 → 수주 선점 + 경쟁사 차단.

## 7. 인수인계 혁신 — 맥락 유실 제로

### 현재 문제

```
PM 이동/퇴직 시:
→ 메일 50통 포워딩 + 구두 인수인계
→ 1~2주 소요
→ 맥락 유실 → 고객 불만 ("전 담당자한테 다 얘기했는데 또 설명해야 해요?"")
```

### 웹 포털 도입 후

```
모든 서비스 이력이 웹에 기록되어 있음
→ 새 PM이 해당 선박 페이지 열면 전체 맥락 파악
→ 메시지 이력, 리포트, 점검 기록, 수리 이력 전부 열람
→ AI가 "이 고객 요약 카드" 자동 생성
→ 인수인계 30분 이내 완료
```

| | 현재 | 웹 포털 후 |
|---|---|---|
| 인수인계 기간 | 1~2주 | **30분~1시간** |
| 맥락 유실 | 높음 (구두 전달) | **제로** (시스템에 전부 있음) |
| 고객 불편 | 높음 ("또 설명해야 해") | **없음** (새 PM이 이미 알고 있음) |

## 8. 서비스 브랜드 가치 — 중고선 가치까지 올린다

### 현재

- AS를 했다 → 끝. 서비스 품질에 대한 객관적 증거 없음
- 선박 매각 시 정비 이력을 증명할 방법이 체계적이지 않음

### 웹 포털이 만드는 서비스 브랜드

| 가치 | 설명 |
|------|------|
| **인증된 점검 이력** | 웹 포털에 3~5년치 정기 점검 기록이 체계적으로 축적 |
| **시각화 건강도 추이** | 배전반 건강도가 지속적으로 관리되었다는 그래프 증거 |
| **서비스 이력 인증서** | "HD Marine Solution Tech Certified" — 정기 점검 이력 인증 |
| **중고선 가치 향상** | 매각 시 "이 선박은 인증된 정기 점검을 받았습니다" 증명 가능 |

> **선주 입장: 중고선 매각 시 정비 이력이 있으면 더 높은 가격을 받을 수 있다.** → 우리 서비스를 받을 인센티브가 생긴다.

## 9. 하도급법 리스크 제거 — 시스템이 자동 차단

### 현재 리스크

```
긴급공사 발생 → 구두로 협력업체에 착수 지시 → 공사 진행
→ PO(발주서) 나중에 발행 → 수기발주서 누락 → 하도급법 위반 리스크
```

- 현재 PM 기억에 의존
- 긴급할수록 절차 누락 가능성 높음
- 하도급법 위반 시: 과징금 + 입찰 제한

### 웹 포털 + AI 자동화 후

```
긴급공사 발생 → 시스템에 등록
→ AI가 "PO 미확인 + 공사 착수" 감지
→ 자동 알림: "수기발주서가 필요합니다"
→ 수기발주서 드래프트 자동 생성
→ PM은 확인 + 서명만
→ 하도급법 위반 리스크 제로
```

## 10. 그룹 전략 정합 — HD현대 AX 전략과 일치

### HD현대 그룹 디지털 전환 방향

- **HD현대마린솔루션**: Hi-4S (24/7 스마트케어), ISS 2.0, OceanWise, 디지털 관제센터
- 그룹 차원의 **AX(AI Transformation)** 전략 추진 중
- 자회사인 우리가 웹 포털을 구축하면 → 그룹 전략과 완전히 정합

### 시너지

| 연동 대상 | 효과 |
|----------|------|
| **Hi-4S** | 엔진/주요기기 모니터링(Hi-4S) + 배전반 모니터링(웹 포털) = 선박 전체 커버 |
| **ISS 2.0** | 통합 스마트십 데이터 + 배전반 데이터 = 더 정밀한 분석 |
| **디지털 관제센터** | 관제센터 데이터 → 웹 포털 대시보드에 연동 → 고객에게도 공개 |

> **Hi-4S는 엔진/주요기기를 커버한다. 배전반은 빈 영역이다.** 우리가 그 빈 영역을 채우면 그룹 차원의 "선박 토탈 케어" 완성.
