# 서론 — 왜 필요한가

## 1. 현재 기술 지원 방식의 한계

우리 회사의 현재 기술 지원 프로세스는 **"전화/메일 → 엔지니어 직접 방문"** 구조입니다.

```
선주 문의 전화 → PM이 상황 파악 → 엔지니어 출장 스케줄 → 이동 → 현장 도착 → 진단 → 조치
```

이 구조의 문제:

| 문제 | 상세 |
|------|------|
| **대응 시간 느림** | 전화 받고 → 엔지니어 배정 → 출장 → 도착까지 최소 수일. 해외라면 수주 |
| **출장 비용 부담** | 항공료, 숙박, 일당. 해외 출장 1건에 수백만 원. 무상 헬스체크라면 전부 우리 부담 |
| **전화로는 한계** | "이 부분에서 소리가 나요" → 전화로는 정확한 상황 파악 불가. 사진 몇 장으로도 부족 |
| **엔지니어 가용성** | 동시에 여러 건 발생하면 → 인원 부족 → 대기 발생 |
| **글로벌 대응 한계** | 선박은 전 세계에 있음. 중동, 유럽, 남미 선박 → 엔지니어가 가려면 시간+비용 엄청남 |
| **간단한 문제도 출장** | 현장 가보니 "스위치 하나 리셋하면 되는 거였다" → 출장비 낭비 |

### 핵심 질문

> "엔지니어가 현장에 가지 않고도, 현장의 눈과 귀가 될 수 있다면?"

## 2. 선도기업은 이미 원격 지원을 하고 있다

### 바르질라 — Smart Support Centre + AR 원격 가이드

바르질라는 2018년부터 AR(증강현실) 기반 원격 지원을 도입했고, 2020년에 **Smart Support Centre**를 정식 런칭했습니다.

**작동 방식:**

```
선상에서 문제 발생
    ↓
선원이 스마트 글래스(또는 태블릿/스마트폰) 착용
    ↓
카메라 + 마이크로 현장 상황 실시간 전송
    ↓
바르질라 전문가가 원격에서 실시간으로 봄
    ↓
전문가가 음성 지시 + 화면에 AR 마커로 "이 부분을 확인하세요" 표시
    ↓
선원이 가이드에 따라 조치
    ↓
문제 해결 (엔지니어 출장 없이)
```

**바르질라 실제 성과:**

| 지표 | 수치 | 출처 |
|------|------|------|
| **원격 해결률** | **95%** — 거의 모든 오류를 원격으로 해결 | Wärtsilä Smart Support Centre |
| **대응 시간** | 즉시 (24/7 전문가 대기) | — |
| **AR 테스트 결과** | WiFi 안테나 + 스마트 글래스로 선상 어디서든 실시간 통신 검증 | 2019년 테스트 |

> 바르질라 공식 발표: "95%의 오류를 원격으로 해결할 수 있습니다. 다음 항구에서 서비스 엔지니어를 부를 필요가 없습니다."

### 콩스버그 — Remote Services (원격 진단/원격 가상 서비스)

콩스버그는 여러 단계의 원격 서비스를 제공합니다:

| 서비스 | 내용 |
|--------|------|
| **Remote Diagnostics (1st Line)** | 실시간으로 선상 시스템 상태를 원격 확인 → 즉석 진단 + 트러블슈팅 |
| **Remote Virtual Services** | 모바일 기기를 통해 선상 인력과 콩스버그 전문가가 실시간 협업 → 트러블슈팅, 운영 가이드, 원격 검사 |
| **Remote Configuration** | 원격으로 시스템 설정 변경, 진단, 소프트웨어 업데이트 |

콩스버그의 핵심 원칙:
> "첫 시도에 해결한다(resolve on first attempt). 그래야 비용과 다운타임을 최소화할 수 있다."

모든 통신은 **PKI 기반 암호화**로 보안 처리.

## 1. 우리 회사에 맞는 원격 지원 시스템 설계

│  ┌──────────┐                  ┌──────────────┐         │
│  │ 스마트폰  │ ──── 영상 ────→ │  전문가 화면  │         │
│  │ 또는      │ ←── 가이드 ──── │  (웹 대시보드) │         │
│  │ 태블릿    │                  │              │         │

### 서비스 단계 — 3단계 대응 체계

#### Level 1. 전화/채팅 원격 지원 (즉시 도입 가능)

> 지금 당장 할 수 있는 것. 추가 투자 거의 없음.

| 항목 | 내용 |
|------|------|
| **방식** | 전화 + 카카오톡/WhatsApp/Teams 영상통화 |
| **절차** | 선원이 스마트폰으로 배전반 촬영 → 우리 엔지니어가 보면서 음성 가이드 |
| **도구** | 기존 스마트폰 + 기존 메신저 앱 (별도 투자 없음) |
| **한계** | 화면 공유, AR 마킹, 녹화/기록 기능 없음 |
| **적합 상황** | 간단한 문의, 스위치 조작 가이드, 알람 확인, 기본 트러블슈팅 |

#### Level 2. 화상/화면 공유 원격 지원 (3~6개월 내 도입)

> 전문 원격 지원 플랫폼 도입. 기록과 품질이 올라감.

| 항목 | 내용 |
|------|------|
| **방식** | 전용 원격 지원 플랫폼 (Microsoft Teams / Zoom + 원격 지원 앱) |
| **절차** | 선원이 앱 실행 → 우리 전문가에게 연결 → 실시간 영상 + 화면 공유 |
| **추가 기능** | 화면 캡처/녹화 (기록용), 포인터로 위치 지시, 파일(도면/매뉴얼) 실시간 전송 |
| **기록** | 세션 녹화 → 서비스 이력에 자동 첨부 → 웹 포털 연동 |
| **적합 상황** | 중간 난이도 트러블슈팅, 측정 가이드, 점검 절차 안내 |

#### Level 3. AR 원격 가이드 (12개월~ 도입)

> 바르질라 Smart Support Centre 수준. 가장 강력한 원격 지원.

| 항목 | 내용 |
|------|------|
| **방식** | AR 글래스(스마트 글래스) 또는 태블릿 AR 앱 |
| **절차** | 선원이 AR 기기 착용 → 우리 전문가에게 연결 → 전문가가 선원의 시야를 실시간으로 봄 |
| **AR 기능** | 전문가가 선원 시야에 **화살표, 원, 텍스트를 AR로 표시** ("이 볼트를 풀어주세요" → 해당 볼트 위에 화살표 표시) |
| **핸즈프리** | 선원은 두 손으로 작업하면서 글래스로 가이드를 받음 (스마트폰 들고 있을 필요 없음) |
| **추가 기능** | 매뉴얼/도면 AR 오버레이, 3D 모델 표시, 세션 전체 녹화 |
| **적합 상황** | 복잡한 수리 작업, 부품 교체 가이드, 신입 선원 지원, 긴급 대응 |

## 2. 구체적 운영 방식

### 원격 지원 요청 프로세스

```
선상에서 문제 발생
    ↓
① 웹 포털 또는 앱에서 "원격 지원 요청" 클릭
   (긴급도 선택: 일반 / 긴급)
    ↓
② 우리 엔지니어에게 자동 배정 + 알림
   (긴급이면 당직 엔지니어 즉시 알림)
    ↓
③ 엔지니어가 원격 세션 시작
   - Level 1: 전화/영상통화
   - Level 2: 화상 공유 플랫폼
   - Level 3: AR 세션
    ↓
④ 실시간 진단 + 가이드
   - 선원 카메라로 현장 확인
   - 엔지니어가 음성/화면으로 지시
   - 필요 시 도면/매뉴얼 전송
    ↓
⑤ 결과 기록
   - 세션 녹화 / 스크린샷 저장
   - 진단 내용 + 조치 사항 기록
   - 웹 포털 서비스 이력에 자동 등록
    ↓
⑥ 후속 조치 판단
   - 원격 해결 완료 → 종료
   - 현장 방문 필요 → 정확한 상황 파악 완료된 상태에서 출장 (헛걸음 방지)
```

### 운영 체계

| 항목 | 내용 |
|------|------|
| **운영 시간** | 초기: 업무 시간 (09:00~18:00 KST). 확대 후: 24/7 |
| **대응 인원** | 배전반 전문 엔지니어 2~3명 교대 (기존 인력 활용) |
| **대응 시간 목표** | 일반: 4시간 이내 세션 시작 / 긴급: 1시간 이내 |
| **지원 언어** | 한국어, 영어 |
| **보안** | 영상 통화 암호화, 녹화 데이터 사내 서버 저장 |

## 3. 도입 시 우리 회사에 돌아오는 이득

### 이득 1. 출장 비용 대폭 절감

| | 현재 (모든 건 출장) | 원격 지원 도입 후 |
|---|---|---|
| **간단한 문의** | 엔지니어 출장 (수백만 원) | 원격 10분에 해결 (0원) |
| **중간 트러블슈팅** | 출장 → 현장 진단 → 조치 | 원격 진단 → 70% 원격 해결, 30% 출장 |
| **복잡한 수리** | 출장 (가봐야 뭔지 앎) | 원격으로 사전 진단 완료 → **정확한 준비 후 출장** |

**비용 절감 시뮬레이션:**
```
연간 출장 건수: 100건 (가정)

현재: 100건 × 평균 출장비 ₩3M = ₩300M

원격 도입 후:
  - 원격 해결: 50건 × ₩0 = ₩0
  - 사전 진단 후 출장: 30건 × ₩3M = ₩90M
  - 기존 방식 출장: 20건 × ₩3M = ₩60M
  - 합계: ₩150M

연간 절감액: ₩150M (50% 절감)
```

### 이득 2. 대응 속도 혁신

| | 현재 | 원격 지원 도입 후 |
|---|---|---|
| 국내 선박 | 1~3일 (출장 준비 + 이동) | **수 시간 이내** (원격 세션 즉시) |
| 해외 선박 | 1~2주 (비자, 항공, 이동) | **수 시간 이내** (원격) |
| 긴급 상황 | "최대한 빨리 가겠습니다" | **1시간 이내 원격 대응 시작** |

**핵심:** 선박은 전 세계에 있지만, 원격 지원은 **거리와 무관하게 동일한 속도**.

### 이득 3. 엔지니어 생산성 향상

| | 현재 | 원격 지원 도입 후 |
|---|---|---|
| 엔지니어 1명 주간 처리 | 출장 1~2건 (이동 시간 포함) | 원격 5~10건 + 필요 시 출장 1건 |
| 이동 시간 | 전체 업무의 30~50% | **0%** (원격 건) |
| 동시 대응 | 불가 (1명이 1곳에) | **순차 대응 가능** (원격이라 바로 다음 건) |

**같은 인원으로 3~5배 더 많은 건을 처리할 수 있습니다.**

### 이득 4. 헛걸음 방지 + 정확한 출장 준비

| | 현재 | 원격 지원 도입 후 |
|---|---|---|
| 출장 전 상황 파악 | 전화로 대충 파악 → 가봐야 앎 | **영상으로 정확히 파악 완료** |
| 부품/공구 준비 | 추정으로 준비 → 현장에서 "이것도 필요했네" | **원격 진단 후 정확한 준비 목록** |
| 헛걸음 | "가보니 스위치 리셋이면 됐는데..." | 원격으로 먼저 확인 → **갈 필요 없는 건은 안 감** |

### 이득 5. 서비스 기록 체계화

| | 현재 | 원격 지원 도입 후 |
|---|---|---|
| 기술 지원 기록 | 전화 통화 → 기록 없음 | **세션 녹화 + 스크린샷 + 조치 기록 자동 저장** |
| 인수인계 | "이 선박 뭐가 문제였더라..." | 과거 원격 세션 녹화본 열람 → **즉시 파악** |
| 노하우 축적 | 엔지니어 머릿속 | 원격 세션이 **교육 자료**가 됨 (신입 학습용) |

### 이득 6. 고객 경험 혁신 — 경쟁사 차별화

| | 현재 (경쟁사도 동일) | 원격 지원 도입 후 |
|---|---|---|
| 고객 경험 | "전화했더니 출장 잡는다고 기다리래" | **"바로 연결되서 화면 보면서 해결해줬다"** |
| 차별화 | 없음 (다 비슷) | **"이 회사는 원격으로 바로 봐준다"** |
| 경쟁 우위 | 가격 경쟁 | **서비스 속도/품질 경쟁** |

---

## 4. 구체적 도입 프로세스

### Step 1. Level 1 즉시 도입 (1~2개월)

| 활동 | 상세 |
|------|------|
| **내부 준비** | 원격 지원 전담 엔지니어 2~3명 지정 (기존 인력) |
| **도구** | 기존 스마트폰 + 영상통화 앱 (Teams/WhatsApp) |
| **프로세스 수립** | 원격 지원 요청 → 대응 → 기록 프로세스 표준화 |
| **고객 안내** | 주요 고객에게 "원격 기술 지원 서비스 시작" 안내 |
| **기록** | 원격 세션 후 간단 보고서 작성 → 서비스 이력 등록 |
| **비용** | ₩0 (추가 투자 없음) |

### Step 2. Level 2 화상 플랫폼 도입 (3~6개월)

| 활동 | 상세 |
|------|------|
| **플랫폼 선정** | 원격 지원 전용 플랫폼 도입 (VSight, TeamViewer, Microsoft Remote Assist 등) |
| **기능** | 화면 공유, 포인터, 캡처/녹화, 파일 전송 |
| **웹 포털 연동** | 원격 세션 기록이 웹 포털 서비스 이력에 자동 등록 |
| **교육** | 엔지니어 + 주요 고객 선원 대상 사용법 교육 |
| **파일럿** | 주요 고객 5~10곳 대상 파일럿 운영 + 피드백 |
| **비용** | 낮음 (플랫폼 라이선스 월 수십만 원) |

### Step 3. Level 3 AR 도입 (12개월~)

| 활동 | 상세 |
|------|------|
| **AR 기기 선정** | RealWear Navigator, Microsoft HoloLens, 또는 태블릿 AR 앱 |
| **파일럿** | 라이프사이클 계약 Premium 고객 대상 AR 기기 배치 |
| **콘텐츠 제작** | 배전반 주요 작업 AR 가이드 콘텐츠 (차단기 교체 절차, 측정 방법 등) |
| **확대** | 성과 확인 후 대상 확대 |
| **비용** | 중간 (AR 기기 대당 ₩1~3M + 플랫폼 비용) |

### 도입 타임라인

```
[즉시~2개월]  Level 1 — 전화/영상통화 원격 지원 시작 (비용 ₩0)
     ↓
[3~6개월]    Level 2 — 화상 플랫폼 도입 + 웹 포털 연동
     ↓
[6~12개월]   Level 2 확대 + 성과 분석
     ↓
[12개월~]    Level 3 — AR 원격 가이드 파일럿
```

# 결론 — 왜 지금 해야 하는가

## 1. 선도기업은 이미 운영 중이다

- **바르질라**: Smart Support Centre로 24/7 원격 지원 운영 중. **95%를 원격 해결**.
- **콩스버그**: Remote Diagnostics + Remote Virtual Services 운영 중. "첫 시도에 해결" 원칙.
- **미/캐나다 해군**: AR 기반 원격 정비를 **실전 배치** 완료. 수리 속도 47% 향상.

**이건 미래 기술이 아니라, 이미 작동하고 있는 현재 기술입니다.**

## 2. Level 1은 내일 시작할 수 있다

- 스마트폰 + 영상통화 = **추가 투자 0원**
- 기존 엔지니어가 전화로 하던 걸 **영상통화로 바꾸기만** 하면 됨
- 이것만 해도 간단한 건의 **50% 이상을 출장 없이 해결** 가능

## 3. 비용 절감이 즉각적이다

- 출장 100건 중 50건만 원격 해결해도 → **연간 ₩150M 절감** (가정)
- 같은 인원으로 **3~5배 더 많은 건 처리** 가능
- 해외 선박 대응이 **수주 → 수 시간**으로 단축

## 4. 라이프사이클 계약과 시너지

- Tier 2 Standard: "기술 지원 무제한" → 원격 지원으로 제공하면 비용 효율적
- Tier 3 Premium: "원격 기술 지원 포함" → **계약의 프리미엄 가치**가 됨
- 원격 세션 기록 → **웹 포털 서비스 이력에 자동 축적** → 데이터 자산

## 5. 한 문장 요약

> **"엔지니어를 보내는 게 아니라, 엔지니어의 눈과 귀를 보낸다.**
> **바르질라는 95%를 원격으로 해결하고 있다.**
> **우리는 스마트폰 하나로 내일부터 시작할 수 있다.**
> **시작하는 데 드는 비용은 0원이다."**
