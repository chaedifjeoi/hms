# 라이프사이클 계약 & 원격 기술 지원 — 현황 분석 및 고도화 방안

## HD현대마린솔루션테크 현재 상태 점검 + 선도기업 벤치마킹 기반 실행 제안

---

# 1. 현재 우리 회사가 이미 하고 있는 것 vs 안 하고 있는 것

## 1-1. 라이프사이클 계약

| 항목 | 현재 상태 | 비고 |
|------|----------|------|
| 무상 헬스체크 (정기 점검) | **이미 하고 있음** | 다만 "무상"이라 수익 0원 |
| 건별 AS/수리 | **이미 하고 있음** | 일회성 수익 구조 |
| 웹 포털 | **구축 중/계획 중** | 별도 설계 문서 존재 |
| **티어별 유상 계약 (Tier 1/2/3)** | **안 하고 있음** | 핵심 제안 사항 |
| 전담 PM 배정 | **안 하고 있음** | |
| SLA (응답 시간 보증) | **안 하고 있음** | |
| 부품 수명 관리/선제 알림 | **안 하고 있음** | |
| AI 분석 리포트 | **안 하고 있음** | |
| 수리 비용 정액제 포함 | **안 하고 있음** | |

**핵심 판단:**
라이프사이클 계약은 **"이미 하고 있는 걸 발표하는 게 아니라, 이미 하고 있는 걸 돈 받는 구조로 바꾸자는 제안"**이다.
헬스체크는 하고 있지만 **무상**이고 **계약 형태가 아님**. 바르질라처럼 이걸 유상 장기 계약으로 패키징하자는 게 핵심.

---

## 1-2. 원격 기술 지원

| 항목 | 현재 상태 | 비고 |
|------|----------|------|
| 전화 기술 상담 | **하고 있음** | "전화/메일 → 엔지니어 방문"이 현재 구조 |
| 영상통화 원격 진단 (Level 1) | **비공식적으로 할 수도 있음** | 공식 체계는 없음 |
| 전용 화상 플랫폼 (Level 2) | **안 하고 있음** | 녹화/기록/포털 연동 없음 |
| AR 원격 가이드 (Level 3) | **안 하고 있음** | |
| 원격 세션 기록 체계화 | **안 하고 있음** | "전화 통화 → 기록 없음" 상태 |
| 원격 지원 요청 프로세스 | **안 하고 있음** | 표준화된 요청→배정→대응 절차 없음 |

**핵심 판단:**
전화로 "이거 어떻게 해요?" 수준의 비공식 상담은 하고 있겠지만, **체계화된 원격 기술 지원 서비스는 안 하고 있음.**
제안 문서 자체가 "전화로는 한계"라고 현재 상태의 문제점을 지적하고 있음.

# 2. 이미 하고 있는 걸 발표해도 되는가?

## 결론: 된다. 오히려 더 좋다.

### 이유 1. "이미 하고 있다"가 아니라 "이미 하고 있는데 돈을 못 받고 있다"가 포인트

- 무상 헬스체크 → **유상 계약화**
- 비공식 전화 상담 → **체계화된 원격 지원 서비스**

### 이유 2. 선도기업 벤치마킹의 본질은 "우리도 할 수 있다"를 보여주는 것

- "바르질라는 이걸 하는데 우리는 바닥부터 시작해야 합니다" → 비현실적으로 보임
- "바르질라는 이걸 하는데 **우리는 이미 기초가 있으니 바로 고도화할 수 있습니다**" → 실행 가능성이 높아 보임

### 이유 3. 발표 프레이밍

> "우리는 이미 헬스체크와 기술 상담을 하고 있습니다.
> 문제는 이게 **무상**이고, **체계가 없고**, **수익으로 연결되지 않는다**는 겁니다.
> 선도기업은 같은 걸 하면서도 **장기 계약 + 체계적 서비스**로 만들어서 안정적 반복 매출을 만들고 있습니다.
> 우리는 이미 기반이 있으니, **포장과 구조만 바꾸면** 바로 시작할 수 있습니다."

# 3. 고도화 방안 — 구체적으로 뭘 해야 하는가

## A. 라이프사이클 계약 고도화

### 1단계: 무상 헬스체크의 유상화 (가장 시급)

| 현재 | 고도화 | 구체적 액션 |
|------|--------|------------|
| 무상으로 점검해주고 끝 | Tier 1 Basic 계약에 포함 | 연간 고정 요금제로 전환. "무상"이 아니라 "계약 서비스"로 포지셔닝 |
| 점검 리포트가 단순 텍스트 | 시각화 리포트 (그래프/대시보드) | 점검 데이터를 그래프, 추이 차트로 시각화. AI 코멘트 추가 |
| 점검 후 연락 없음 | 자동 리마인더 + 교체 시기 알림 | 부품별 수명 추적 시스템 구축 → 선제적 알림 발송 |

### 2단계: 계약 구조 설계 및 파일럿

| 액션 | 상세 |
|------|------|
| **과거 데이터 분석** | 기존 고객별 AS 건수, 비용, 빈도를 분석해서 Tier별 가격 산정 근거 마련 |
| **계약서 템플릿** | Tier 1/2/3 각각의 계약서 초안 작성. 서비스 범위, SLA, 가격 구조 명확히 |
| **파일럿 고객 3곳** | 관계 좋고 선박 많은 선주 선정. Tier 1부터 시작 |
| **성과 지표 설계** | 갱신율, 업셀률, 고객 만족도, 반복 매출 비중 등 KPI 정의 |

### 3단계: 웹 포털과 연계

| 액션 | 상세 |
|------|------|
| **계약 고객 전용 포털 기능** | 호선정보, 점검 이력, 리포트 열람, 서비스 요청 |
| **비계약 고객과 차별화** | 계약 고객만 포털 접근 가능 → 계약의 가치를 체감하게 함 |
| **데이터 축적 → Lock-in** | 3년간 쌓인 데이터는 경쟁사가 못 가짐 → 고객 이탈 방지 |

---

## B. 원격 기술 지원 고도화

### 1단계: 즉시 도입 (0~2개월, 비용 0원)

| 액션 | 상세 |
|------|------|
| **영상통화 공식화** | "전화만 받던 것"을 "영상통화 원격 진단 서비스"로 공식 런칭 |
| **전담 인원 지정** | 엔지니어 2~3명을 원격 지원 담당으로 지정 (기존 인력) |
| **프로세스 표준화** | 요청 접수 → 배정 → 영상 연결 → 진단 → 기록의 흐름을 문서화 |
| **간단 기록 양식** | 원격 세션 후 "어떤 문제 → 어떻게 해결 → 후속 필요 여부" 기록 |
| **고객 안내** | 주요 고객에게 "원격 기술 지원 서비스 시작합니다" 공문/안내 |

### 2단계: 플랫폼 도입 (3~6개월)

| 액션 | 상세 |
|------|------|
| **전용 플랫폼 도입** | VSight, TeamViewer, MS Remote Assist 중 선정 |
| **핵심 기능** | 화면 공유 + 포인터 지시 + 캡처/녹화 + 파일 전송 |
| **웹 포털 연동** | 원격 세션 기록이 포털의 서비스 이력에 자동 등록 |
| **KPI 측정 시작** | 원격 해결률, 평균 대응 시간, 출장 절감 건수 추적 |

### 3단계: AR + 라이프사이클 계약 연계 (12개월~)

| 액션 | 상세 |
|------|------|
| **AR 파일럿** | Premium 계약 고객 대상 AR 글래스 배치 |
| **계약 연계** | Tier 2에 "원격 지원 무제한" 포함, Tier 3에 "AR 원격 가이드" 포함 |
| **원격 해결률 목표** | 바르질라 95% 벤치마크 → 우리는 1년차 50%, 2년차 70% 목표 |

## C. 두 가지를 합쳐서 시너지를 내는 방법

이것이 발표에서 **가장 강력한 포인트**가 될 수 있음.

### 시너지 구조

```
라이프사이클 계약 + 원격 기술 지원 = 시너지

┌─────────────────────────────────────────────────────────┐
│                                                          │
│  Tier 2 Standard 계약                                    │
│  → "원격 기술 지원 무제한" 포함                          │
│  → 고객: "전화하면 바로 영상으로 봐준다" → 계약 가치 체감 │
│  → 우리: 출장 안 가고 원격 해결 → 비용 절감 → 마진 개선  │
│                                                          │
│  Tier 3 Premium 계약                                     │
│  → "AR 원격 가이드" 포함                                 │
│  → 계약의 프리미엄 가격 정당화                           │
│  → 경쟁사 대비 명확한 차별화 포인트                      │
│                                                          │
│  원격 세션 기록 → 웹 포털 서비스 이력 축적               │
│  → 데이터가 쌓일수록 우리만의 자산 → 고객 Lock-in 강화   │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 수익 시뮬레이션 (두 가지 결합 시)

```
[매출 증가]
Tier 2 Standard × 50척 × ₩20M/년 = ₩10억 반복 매출

[비용 절감]
원격 지원으로 출장 50% 감소 = 연간 ₩1.5억 절감

[합산 효과]
연간 ₩10억 신규 반복 매출 + ₩1.5억 비용 절감
= 사실상 ₩11.5억 이상의 순효과

+ 고객 이탈 방지 (Lock-in)
+ 업셀 기회 (Tier 1 → 2 → 3)
+ 데이터 자산 축적
```

# 4. 발표 시 추천 프레이밍

## 발표 스토리라인

```
① 현재 문제 인식
   "우리는 건별 일회성 수익 구조에 의존하고 있다"
   "헬스체크를 무상으로 하고 있지만, 수익으로 연결되지 않는다"
   "기술 지원은 전화 + 출장 중심이라 느리고 비용이 크다"

           ↓

② 선도기업 벤치마킹
   "바르질라는 같은 서비스를 장기 계약으로 묶어서 반복 매출을 만들고 있다"
   "바르질라는 95%를 원격으로 해결하고 있다"
   "이건 미래 기술이 아니라, 이미 표준이 되고 있는 현재 트렌드다"

           ↓

③ 우리의 유리한 위치
   "우리는 이미 헬스체크를 하고 있다 → 계약화 기반이 있다"
   "우리는 HD현대 건조 선박의 유일한 AS 사업자 → 독점적 지위"
   "9,890척 네트워크 → 계약 제안할 고객 풀이 이미 있다"

           ↓

④ 고도화 제안
   "무상 헬스체크 → 유상 라이프사이클 계약으로 전환"
   "전화 상담 → 체계적 원격 기술 지원 서비스로 전환"
   "두 가지를 합쳐 시너지를 낸다"

           ↓

⑤ 기대 효과
   "안정적 반복 매출 ₩10억+ 창출"
   "출장 비용 50% 절감"
   "고객 Lock-in + 경쟁사 차별화"
```

## 핵심 메시지

> **"지금 무상으로 하고 있는 것을, 라이프사이클 계약이라는 그릇에 담고,
> 원격 기술 지원이라는 무기를 더하면,
> 같은 역량으로 수익 구조를 완전히 바꿀 수 있다.
> 바르질라는 이미 하고 있고, 우리는 기반이 있으니 바로 시작할 수 있다."**

# 5. 요약 — 한눈에 보기

| 항목 | 라이프사이클 계약 | 원격 기술 지원 |
|------|-----------------|---------------|
| **현재 상태** | 기초 있음 (무상 헬스체크) | 거의 없음 (비공식 전화만) |
| **이미 하고 있는 것 발표?** | OK — "유상화" 제안이 핵심 | OK — 체계화 자체가 새로운 것 |
| **가장 시급한 액션** | 무상 헬스체크 유상 계약 전환 | 영상통화 원격 진단 공식화 |
| **초기 비용** | 낮음 (계약서, 리포트 설계) | 0원 (기존 스마트폰 활용) |
| **기대 효과** | 반복 매출 창출 + 고객 Lock-in | 출장 비용 절감 + 대응 속도 혁신 |
| **선도기업 대비 격차** | Level 2 → Level 4로 올려야 함 | 0단계 → Level 1부터 시작 |
| **시너지** | 원격 지원을 계약에 포함 → 계약 가치 상승 | 계약 고객 대상 우선 제공 → 차별화 |
