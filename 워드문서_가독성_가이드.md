# 가독성 높은 프로페셔널 워드 문서 제작 가이드

> 부서장 발표용 워드 문서를 **표, 그래프, 하이라이트, 그림**을 활용하여 깔끔하고 가독성 있게 만들기 위한 종합 가이드

## 0. 부서장/임원 보고서 — 대원칙 (가장 중요)

> **경영진은 "전체 흐름"과 "핵심 수치"만 본다. 디테일은 필요하면 꺼내 보는 것이지, 처음부터 읽는 게 아니다.**

### 개조식 작성법 (한국 기업 보고서의 표준)

개조식은 문장을 간결하게 줄여 핵심만 전달하는 방식입니다. 임원 보고서의 기본입니다.

| 원칙 | 설명 | 예시 |
|------|------|------|
| **명사형 종결** | '-음', '-임', '-함'으로 끝맺음 | "웹 포털 구축 필요 → 고객 접점 확대 가능" |
| **조사 생략** | 불필요한 조사는 과감히 생략 | "시장 규모 371억 달러(2024) → 532억 달러(2032)" |
| **마침표 없음** | 항목 끝에 마침표 찍지 않음 | "디지털 전환 가속화 중" (O) / "디지털 전환 가속화 중." (X) |
| **위계 명확** | 들여쓰기·번호로 상하 관계 표현 | 1. 대항목 → 1-1. 소항목 → - 세부 내용 |
| **수치 우선** | 정성적 표현보다 정량적 수치 | "많이 증가" (X) → "전년 대비 23% 증가" (O) |

### 임원이 보는 순서 (= 문서 구조 우선순위)

```
┌──────────────────────────────────────────────────┐
│  1순위: Executive Summary (1페이지로 끝내기)      │
│         → 결론 먼저. "뭘 해달라는 건지" 30초 파악  │
├──────────────────────────────────────────────────┤
│  2순위: 핵심 그래프/표 (시각 요소)                │
│         → 시장 규모, Gap 분석, Before/After        │
├──────────────────────────────────────────────────┤
│  3순위: 로드맵/기대효과                           │
│         → 언제 뭘 하고, 뭘 얻는지                 │
├──────────────────────────────────────────────────┤
│  4순위: 상세 내용 (필요할 때만)                   │
│         → 벤치마킹 상세, 기능 설계 등              │
└──────────────────────────────────────────────────┘
```

### "한 페이지 한 메시지" 원칙

| 규칙 | 설명 |
|------|------|
| **1페이지 = 1핵심 메시지** | 한 페이지에 여러 주제를 섞지 않음 |
| **상단 1/3에 결론** | 페이지 상단에 핵심 메시지, 하단에 근거 |
| **So What?** | 모든 데이터 뒤에 "그래서 뭘 해야 하는데?"가 있어야 함 |
| **3의 법칙** | 핵심 포인트는 3개 이내 — 5개 넘으면 기억 못함 |

## 2.5. 비주얼 계층 구조 & 여백 (Visual Hierarchy & Whitespace)

> 가독성의 80%는 "비워두는 것"에서 나온다. 빽빽한 문서는 아무리 좋은 내용이라도 읽히지 않는다.

### 비주얼 계층 7원칙

| 원칙 | 설명 | 적용 |
|------|------|------|
| **크기 (Size)** | 큰 요소가 먼저 눈에 들어옴 | 제목 20pt > 소제목 16pt > 본문 10pt |
| **컬러 (Color)** | 진한/밝은 색이 연한 색보다 먼저 인식 | 핵심 수치는 네이비 Bold, 본문은 다크 그레이 |
| **대비 (Contrast)** | 명도 차이가 클수록 눈에 띔 | 표 헤더(진한 배경+흰 글씨) vs 본문 행(흰 배경) |
| **정렬 (Alignment)** | 일관된 정렬이 안정감 | 본문 좌측 정렬 통일, 숫자는 우측 정렬 |
| **반복 (Repetition)** | 동일 패턴 반복 = 통일감 | 모든 표, 모든 그래프에 동일 스타일 |
| **근접 (Proximity)** | 관련 요소는 가까이 배치 | 그래프와 캡션 사이 간격 최소화 |
| **여백 (Whitespace)** | 빈 공간이 요소를 분리/강조 | 섹션 간 24pt+ 여백, 표 전후 12pt 여백 |

### 여백 가이드

| 위치 | 권장 여백 | 효과 |
|------|-----------|------|
| **페이지 여백** | 상하좌우 2.54cm (임원 보고서는 3cm도 가능) | 답답함 방지, 여유 있는 느낌 |
| **섹션 간** | 24~36pt (약 1~1.5줄) | 주제 전환이 명확하게 느껴짐 |
| **표 전후** | 12pt | 표가 본문에 붙어 보이지 않게 |
| **그래프 전후** | 18pt | 그래프가 돋보이게 |
| **인사이트 박스 전후** | 12pt | 강조 효과 극대화 |
| **불릿 항목 간** | 3~4pt | 항목 구분되되, 너무 벌어지지 않게 |

### 절대 하지 말 것

| 안티패턴 | 왜 나쁜가 | 대안 |
|----------|----------|------|
| 한 페이지에 빈 공간 없이 꽉 채움 | 읽기 전에 지침 | 여백 넉넉히, 내용은 다음 페이지로 |
| 모든 곳에 Bold/하이라이트 | 아무것도 강조되지 않음 | 페이지당 Bold 포인트 3개 이내 |
| 색상 7가지 이상 사용 | 산만, 비전문적 | 4색 팔레트 엄수 |
| 그래프 없이 숫자만 나열 | 패턴/추세를 못 봄 | 3줄 이상의 숫자 → 그래프로 |
| 긴 문장 서술형 | 임원은 안 읽음 | 개조식 + 핵심 키워드 Bold |

## 3. 표(Table) 디자인 — 가독성의 핵심

### 3-1. 표 스타일 원칙

```
┌──────────────────────────────────────────────────┐
│  ✓ DO                    │  ✗ DON'T              │
├──────────────────────────┼───────────────────────┤
│  헤더 행 진한 배경+흰 글씨 │  모든 행 같은 색상     │
│  교대 행 배경색            │  과도한 테두리         │
│  세로 정렬 통일 (가운데)   │  셀 안에 텍스트 꽉 채움 │
│  셀 패딩 여유롭게          │  너무 많은 열 (7개+)   │
│  핵심 셀 Bold 강조        │  셀 병합 남발          │
│  캡션 달기 (표 1. ...)    │  캡션 없이 표만 배치   │
└──────────────────────────┴───────────────────────┘
```

### 3-2. 표 유형별 적용법

| 표 유형 | 언제 사용 | 디자인 포인트 |
|---------|----------|--------------|
| **비교표** (Before/After) | 현재 vs 미래, 경쟁사 비교 | 2열 구조, 왼쪽 회색/오른쪽 블루 배경 |
| **기능 목록표** | 기능 상세 설명 | 기능명 Bold, 설명 일반 텍스트 |
| **Gap 분석표** | 선도기업 vs 우리 | 🔴🟡🟢 아이콘으로 시각적 Gap 표현 |
| **로드맵표** | Phase별 계획 | 단계별 색상 그라데이션 |
| **효과 정리표** | 기대효과 요약 | 아이콘 + 짧은 설명 |

### 3-3. python-docx 표 스타일 코드 예시

```python
from docx import Document
from docx.shared import Pt, Inches, Cm, RGBColor
from docx.enum.table import WD_TABLE_ALIGNMENT
from docx.oxml.ns import qn

def style_table_header(table):
    """표 헤더 행: 진한 네이비 배경 + 흰색 Bold 글씨"""
    header_row = table.rows[0]
    for cell in header_row.cells:
        # 배경색 설정
        shading = cell._element.get_or_add_tcPr()
        shading_elm = shading.makeelement(qn('w:shd'), {
            qn('w:fill'): '1B3A5C',
            qn('w:val'): 'clear'
        })
        shading.append(shading_elm)
        # 글씨 흰색 Bold
        for paragraph in cell.paragraphs:
            for run in paragraph.runs:
                run.font.color.rgb = RGBColor(0xFF, 0xFF, 0xFF)
                run.font.bold = True
                run.font.size = Pt(10)

def style_alternating_rows(table):
    """교대 행 배경색: 연한 블루 / 흰색"""
    for i, row in enumerate(table.rows[1:], 1):
        if i % 2 == 0:
            for cell in row.cells:
                shading = cell._element.get_or_add_tcPr()
                shading_elm = shading.makeelement(qn('w:shd'), {
                    qn('w:fill'): 'F0F4F8',
                    qn('w:val'): 'clear'
                })
                shading.append(shading_elm)
```

## 4. 그래프/차트 삽입 — "숫자를 그림으로"

### 4-1. 그래프 유형별 사용법

| 데이터 유형 | 추천 그래프 | 예시 |
|------------|------------|------|
| **시간 추이** | 꺾은선 그래프 (Line) | 시장 규모 성장 추이, 절연저항 변화 |
| **비교** | 가로 막대 차트 (Bar) | 선도기업 vs 우리 기능 비교 |
| **비율/구성** | 도넛 차트 (Donut) | Gap 유형 분포, 기능별 비중 |
| **진행률** | 프로그레스 바 | 로드맵 Phase별 진행률 |
| **상태 표시** | 게이지 차트 | 건강도 점수 (100점 만점) |
| **프로세스** | 플로우 다이어그램 | 서비스 요청 → 완료 흐름 |

### 4-2. matplotlib로 그래프 생성 → 워드 삽입

```python
import matplotlib.pyplot as plt
import matplotlib
matplotlib.rcParams['font.family'] = 'Malgun Gothic'  # 한글 폰트

def create_market_growth_chart(save_path):
    """시장 규모 성장 추이 꺾은선 그래프"""
    years = [2024, 2025, 2026, 2027, 2028, 2029, 2030, 2031, 2032]
    market = [371, 389, 408, 428, 449, 470, 493, 517, 532]

    fig, ax = plt.subplots(figsize=(8, 4))
    ax.plot(years, market, color='#1B3A5C', linewidth=2.5, marker='o',
            markerfacecolor='#2B579A', markersize=8)
    ax.fill_between(years, market, alpha=0.1, color='#2B579A')
    ax.set_title('글로벌 선박 수리·보수 시장 규모 (억 달러)', 
                 fontsize=14, fontweight='bold', color='#1B3A5C')
    ax.set_ylabel('억 달러')
    ax.grid(axis='y', alpha=0.3)
    ax.spines['top'].set_visible(False)
    ax.spines['right'].set_visible(False)
    plt.tight_layout()
    plt.savefig(save_path, dpi=200, bbox_inches='tight')
    plt.close()

# 워드에 삽입
# doc.add_picture(save_path, width=Inches(6))
```

## 5. 하이라이트 & 강조 기법

### 5-1. 강조 수준별 적용법

| 강조 수준 | 기법 | 사용 빈도 | 예시 |
|----------|------|----------|------|
| **★★★ 최고 강조** | 컬러 박스 (음영 배경 + Bold) | 섹션당 1~2개 | 핵심 인사이트, 결론 |
| **★★ 중간 강조** | Bold + 포인트 컬러 텍스트 | 적당히 | 핵심 키워드, 숫자 |
| **★ 약한 강조** | Bold만 | 자유롭게 | 항목명, 중요 단어 |
| **인용/발췌** | 왼쪽 파란 보더 + 회색 배경 | 섹션당 1개 | 핵심 메시지 |

### 5-2. 핵심 인사이트 박스 (Callout Box)

```python
def add_callout_box(doc, text, box_color='#E8F0FE', border_color='#1B3A5C'):
    """파란색 왼쪽 보더 + 연한 배경의 인사이트 박스"""
    table = doc.add_table(rows=1, cols=1)
    cell = table.cell(0, 0)
    cell.text = text

    # 셀 배경색
    tcPr = cell._element.get_or_add_tcPr()
    shading = tcPr.makeelement(qn('w:shd'), {
        qn('w:fill'): box_color.replace('#', ''),
        qn('w:val'): 'clear'
    })
    tcPr.append(shading)

    # 왼쪽 보더 굵게
    borders = tcPr.makeelement(qn('w:tcBorders'), {})
    left = borders.makeelement(qn('w:left'), {
        qn('w:val'): 'single',
        qn('w:sz'): '24',
        qn('w:color'): border_color.replace('#', ''),
        qn('w:space'): '0',
    })
    borders.append(left)
    tcPr.append(borders)

    # Bold 스타일
    for para in cell.paragraphs:
        for run in para.runs:
            run.font.bold = True
            run.font.size = Pt(11)
            run.font.color.rgb = RGBColor(0x1B, 0x3A, 0x5C)
```

```python
def add_horizontal_line(doc, color='#CCCCCC'):
    """섹션 구분용 가로선"""
    paragraph = doc.add_paragraph()
    pPr = paragraph._element.get_or_add_pPr()
    pBdr = pPr.makeelement(qn('w:pBdr'), {})
```

```python
def add_cover_page(doc, title, subtitle, date, author):
    """프로페셔널 표지"""
    # 빈 줄로 상단 여백
    for _ in range(6):
        doc.add_paragraph('')

    # 메인 타이틀
    p = doc.add_paragraph()
    p.alignment = WD_ALIGN_PARAGRAPH.CENTER
    run = p.add_run(title)
    run.font.size = Pt(26)
    run.font.bold = True
    run.font.color.rgb = RGBColor(0x1B, 0x3A, 0x5C)

    # 구분선
    p = doc.add_paragraph()
    p.alignment = WD_ALIGN_PARAGRAPH.CENTER
    run = p.add_run('━' * 30)
    run.font.color.rgb = RGBColor(0x2B, 0x57, 0x9A)
    run.font.size = Pt(12)

    # 부제
    p = doc.add_paragraph()
    p.alignment = WD_ALIGN_PARAGRAPH.CENTER
    run = p.add_run(subtitle)
    run.font.size = Pt(14)
    run.font.color.rgb = RGBColor(0x66, 0x66, 0x66)

    # 날짜 + 작성자
    for _ in range(4):
        doc.add_paragraph('')
    p = doc.add_paragraph()
    p.alignment = WD_ALIGN_PARAGRAPH.CENTER
    run = p.add_run(f'{date}\n{author}')
    run.font.size = Pt(11)
    run.font.color.rgb = RGBColor(0x99, 0x99, 0x99)

    doc.add_page_break()
```

## 9. 숫자 → 시각화 변환 기준

> 숫자가 3줄 이상 나열되면 무조건 그래프로 바꿔라

| 데이터 형태 | 변환 대상 | 예시 |
|------------|----------|------|
| 시간에 따른 변화 | 꺾은선 그래프 + 주석 | 시장 규모 2024→2032 |
| 두 그룹 비교 | 나란히 막대 차트 | 선도기업 vs 우리 역량 |
| 비율/구성 | 도넛 차트 | Gap 유형 분포 |
| 단일 점수 | 게이지 차트 | 건강도 78/100 |
| 순서/단계 | 프로세스 플로우 | 서비스 요청→완료 |
| 핵심 KPI 3~4개 | KPI 카드 (컬러 박스) | 9,890척, 371억$, 4.6% |
