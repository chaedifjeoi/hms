# 선도기업 AM 전략 벤치마킹을 통한 HD현대마린솔루션테크의 미래 방향 제안 (v2)

---

## 1. AM(Aftermarket) 시장 현황

### 시장 규모

- 글로벌 선박 수리·보수 시장: **2024년 371억 달러(55조 원)** → **2032년 532억 달러(78조 원)** (연평균 4.6% 성장)
- 해양 소프트웨어 시장: 2024년 29.9억 달러 → 2033년 78.4억 달러 (연평균 11.3% 성장)

### 핵심 트렌드

- **디지털 전환**: OEM들이 단순 AS → **웹 기반 디지털 플랫폼**으로 고객 접점 전환
- **고객 셀프서비스**: 전화/메일 → 웹 포털에서 서비스 요청, 이력 조회, 문서 열람
- **데이터 기반 서비스**: 수집된 데이터를 시각화하여 고객에게 분석 리포트 제공
- **플랫폼 통합**: 흩어진 레거시 시스템을 하나의 웹 플랫폼으로 통합하는 추세

## 2. 선도기업 벤치마킹

### 2-1. 바르질라 (Wärtsilä) — "웹 포털에서 모든 서비스를 한다"

**핵심 플랫폼: Wärtsilä Online (웹 포털)**

> https://www.wartsila.com/wartsila-online

바르질라는 **웹 브라우저 기반 고객 포털**을 운영하며, 고객이 24/7 스스로 서비스에 접근할 수 있는 환경을 구축했습니다.

| 기능                          | 상세                                                                            |
| ----------------------------- | ------------------------------------------------------------------------------- |
| **서비스 요청 (TechRequest)** | 웹에서 기술 지원 요청 생성 → 과거 요청 이력 + 답변 조회 가능                    |
| **기술 문서 열람**            | 최신 매뉴얼, 기술 회보(Technical Bulletin) 24/7 웹에서 열람                     |
| **견적 요청/추적**            | 온라인 견적 요청 → 진행 상태 실시간 확인 → 관련 문서(패킹리스트, 인보이스) 조회 |
| **클레임/워런티 관리**        | 부품 클레임, 워런티 요청 등록 → 처리 진행 추적 → 전체 이력 조회                 |
| **호선(설비) 정보 관리**      | 고객이 자사 설비 정보를 웹에서 직접 관리 + 이력 확인                            |
| **전문가 연결**               | 포털 내에서 바르질라 엔지니어에게 직접 문의                                     |

**AM 전략 핵심:**

- 전화/메일로 하던 모든 서비스 요청을 **웹 포털 한 곳**에서 처리
- 고객이 **자기 설비의 서비스 이력을 직접 조회**할 수 있음
- 견적, 클레임, 기술 지원 모두 **요청 → 추적 → 이력 관리**가 웹에서 완결

### 2-2. 콩스버그 (Kongsberg) — "웹 플랫폼으로 선박과 육상을 연결한다"

**핵심 플랫폼: K-IMS (웹 기반) + KM Performance (2026 신규)**

> https://www.kongsberg.com/maritime/products/digital/k_ims_applications/k-ims/

콩스버그는 **웹 브라우저 기반 정보 관리 시스템(K-IMS)**을 중심으로 서비스를 제공하고 있으며, 2026년 통합 플랫폼으로 대대적 리뉴얼했습니다.

| 기능                           | 상세                                                                                  |
| ------------------------------ | ------------------------------------------------------------------------------------- |
| **K-IMS (웹 포털)**            | 선박-육상 간 정보 관리 플랫폼. **웹 브라우저로 접속**, 선상/육상 어디서든 데이터 접근 |
| **정보 흐름 관리**             | 문서, 점검 이력, 기술 데이터를 선박↔육상 간 실시간 동기화                             |
| **서비스 요청**                | 웹에서 지원 요청 + 부품 요청 통합 접수 (support/spare part request 단일 폼)           |
| **글로벌 24/7 지원**           | 34개국 네트워크, 24시간 대응, 웹+전화+메일 병행                                       |
| **KM Performance (2026 신규)** | K-IMS, Vessel Insight, K-Fleet, Coach 등 레거시 시스템을 **하나의 웹 생태계로 통합**  |
| **데이터 분석**                | 항해 계획, 운항 모니터링, 트림 최적화, 배출 리포팅 — 웹 대시보드 제공                 |

**AM 전략 핵심:**

- **웹 브라우저 기반** — 별도 앱 설치 없이 어디서든 접근
- 흩어진 레거시 시스템을 **하나의 통합 웹 플랫폼(Digital Ocean)**으로 재편
- 선박과 육상 간 **데이터 동기화 + 협업 도구** 역할
- 단순 서비스 접수 → **데이터 시각화 + 분석 대시보드**까지 확장

## 4. 제안 — 우리 회사 웹 포털 구축

### 4-1. 웹 포털 핵심 기능 설계

우리 회사가 웹 포털을 만든다면, 다음 기능이 필요합니다:

### A. 호선 정보 조회 (고객 셀프서비스)

| 기능                 | 설명                                            |
| -------------------- | ----------------------------------------------- |
| **선박 검색**        | 선박명/IMO 번호로 검색 → 해당 선박 정보 페이지  |
| **배전반 스펙 조회** | 설치된 배전반 종류, 모델, 설치 연도, 회로도     |
| **기자재 목록**      | 해당 선박에 설치된 우리 담당 기자재 전체 리스트 |
| **설치 이력**        | 최초 설치, 교체, 업그레이드 이력                |

**효과:** 고객이 전화해서 "우리 배 배전반 모델이 뭐였지?" 물어볼 필요 없음

### B. 서비스 요청 + 진행 추적

| 기능                   | 설명                                                          |
| ---------------------- | ------------------------------------------------------------- |
| **온라인 서비스 요청** | 웹에서 작업 요청 접수 (선박 선택 → 작업 내용 입력 → 첨부파일) |
| **자동 PM 배정**       | 요청 접수 → 담당 PM에게 자동 배정 + 알림                      |
| **진행 현황 추적**     | 접수 → 견적 → 승인 → 공사 → 완료 단계별 실시간 표시           |
| **이력 조회**          | 과거 모든 서비스 요청 + 결과를 검색/조회 가능                 |
| **담당자 연락**        | 해당 건 담당 PM/엔지니어에게 웹에서 바로 메시지 or 연락       |

**효과:** "지금 우리 건 어디까지 진행됐어요?" 전화 → 웹에서 직접 확인

### C. 점검/수리 이력 관리

| 기능              | 설명                                                |
| ----------------- | --------------------------------------------------- |
| **헬스체크 이력** | 과거 헬스체크 방문 일자, 점검 항목, 결과 요약       |
| **수리 이력**     | 과거 수리 건별 작업 내용, 교체 부품, 비용           |
| **리포트 열람**   | 과거 발행된 점검 리포트를 웹에서 즉시 열람/다운로드 |
| **타임라인 뷰**   | 해당 선박의 전체 서비스 히스토리를 시간순으로 표시  |

**효과:** 선주가 "작년에 뭘 고쳤더라?" → 웹에서 즉시 확인

### D. 리포트 시각화 대시보드

| 기능                         | 설명                                                   |
| ---------------------------- | ------------------------------------------------------ |
| **배전반 건강도 점수**       | 종합 점수 게이지 차트 (100점 만점)                     |
| **측정값 추이 그래프**       | 절연저항, 차단기 동작시간, 온도 등 시간별 트렌드       |
| **정상/주의/위험 구간 표시** | 기준선과 함께 현재 상태 시각적 표시                    |
| **부품 수명 예측**           | 잔여 수명 프로그레스 바 + 교체 권장 시기               |
| **향후 전망**                | AI 기반 트렌드 예측 그래프 ("이 추세면 N개월 후 위험") |

**효과:** 숫자 나열 리포트 → 그래프/차트 대시보드 → 선주 의사결정 속도 향상

### E. 기술 문서 라이브러리

| 기능              | 설명                                             |
| ----------------- | ------------------------------------------------ |
| **매뉴얼**        | 배전반 운용/정비 매뉴얼 호선별 제공              |
| **단선도/복선도** | 해당 선박 배전반 도면 열람/다운로드              |
| **기술 회보**     | Technical Bulletin (안전 공지, 업데이트 정보 등) |
| **부품 카탈로그** | 적용 부품 목록 + 스펙                            |

**효과:** "도면 보내주세요" 전화 → 웹에서 24/7 셀프서비스

### F. 알림/리마인더

| 기능                    | 설명                                               |
| ----------------------- | -------------------------------------------------- |
| **정기 점검 알림**      | "이 배전반 마지막 점검 1년 경과 — 헬스체크 권장"   |
| **부품 교체 시기 알림** | "ACB 설치 후 5년 경과 — 점검/교체 검토 권장"       |
| **입거 일정 연동**      | 입거 예정일 등록 → 사전 준비 알림 자동 발송        |
| **서비스 진행 알림**    | 견적 발송, PO 수신, 공사 완료 등 단계 변경 시 알림 |

**효과:** 우리가 먼저 알려줌 → 선주 신뢰 → 선제적 수주

### 4-2. 웹 포털 기대 효과

### 고객 관점

| Before (현재)                  | After (웹 포털)        |
| ------------------------------ | ---------------------- |
| 배전반 스펙 확인 → 전화 문의   | 웹에서 **즉시 검색**   |
| 서비스 요청 → 전화/메일        | 웹에서 **24/7 접수**   |
| 진행 현황 → 전화 문의          | 웹에서 **실시간 추적** |
| 과거 이력 → 요청 후 대기       | 웹에서 **직접 조회**   |
| 기술 문서 → 요청 후 수신       | 웹에서 **24/7 열람**   |
| 리포트 → 텍스트 나열           | **그래프/대시보드**    |
| 점검 시기 → 고객이 알아서 관리 | **자동 알림**          |

### 회사 관점

| 효과               | 설명                                                    |
| ------------------ | ------------------------------------------------------- |
| **수주 속도 향상** | 시각화된 리포트 → 선주 빠른 의사결정 → 수주 단축        |
| **업무 부하 감소** | "스펙 알려주세요", "진행 상황 알려주세요" 전화 감소     |
| **선제적 영업**    | 알림/리마인더 → 우리가 먼저 접근 → 수주 기회 확대       |
| **데이터 축적**    | 고객 행동 데이터 → 맞춤형 서비스/마케팅 기반            |
| **고객 충성도**    | 편리한 서비스 경험 → 경쟁사 대비 차별화 → 재이용률 향상 |
| **인수인계 용이**  | 모든 이력이 시스템에 → PM 교체 시 맥락 유실 없음        |

---

### 4-3. 추가 제안 — PM 업무 AI 자동화 (내부 효율화)

웹 포털의 **백엔드를 지원하는 내부 시스템**으로, 동시에 구축하면 시너지가 납니다.

| 기능                    | 설명                                                                    |
| ----------------------- | ----------------------------------------------------------------------- |
| **메일 AI 분석**        | HMS 메일 수신 → Work Scope, 일정, 인원, 견적 자동 추출 (3초)            |
| **건별 카드 자동 생성** | 분석 결과 → 건별 카드로 정리 → 타임라인 자동 기록                       |
| **인수인계 자동화**     | PM 변경 시 AI가 전체 담당 건 요약 문서 자동 생성                        |
| **협력업체/견적 매칭**  | 과거 이력 기반 협력업체 추천 + 러프 견적 자동 산출                      |
| **수기발주서 알림**     | 긴급공사 시 PO 미확인 → 수기발주서 자동 드래프트 + 알림 (하도급법 대응) |

**웹 포털과의 시너지:**

- 웹 포털에서 고객이 서비스 요청 → AI가 자동 분류 → PM에게 정리된 카드로 전달
- 고객이 웹에서 진행 현황 조회 → PM이 업데이트한 내용이 자동 반영
- 리포트를 AI가 자동 생성 → 웹 포털에 업로드 → 고객이 열람
