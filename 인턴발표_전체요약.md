# 인턴 발표 전체 요약

## 발표 제목
> **선도기업 AM 전략 벤치마킹을 통한 HD현대마린솔루션테크의 미래 방향 제안**

## 발표자 배경
- HD현대마린솔루션테크 인턴 (배전반 PM)
- HD현대마린솔루션의 자회사. 배전반/전기자동제어반 제조 + 선박 A/S
- HMS(조선소)로부터 작업 문의 → T/S → 협력업체 견적 → HMS 견적 회신 → PO → 수주 → 발주 워크플로우

---

## 핵심 메시지 (한 줄)
> "선도기업과의 차이는 기술력이 아니라 '디지털 창구'다. 웹 포털 하나가 그 창구가 된다."

---

## 벤치마킹 대상 3사

### 바르질라 (Wärtsilä) — Wärtsilä Online (웹 포털)
- 웹에서 서비스 요청(TechRequest), 기술 문서 24/7 열람, 견적 요청/추적, 클레임/워런티 관리, 설비 정보 관리, 전문가 연결
- 부품 온라인 주문도 가능하지만 **우리는 자재를 HMS 통해 공급하므로 부품 판매 부분은 제외**
- 라이프사이클 계약(Lifecycle Agreement) 적극 추진 — 2024~2026년 수십 척 규모 장기 계약 연이어 체결
- Smart Support Centre: 원격 지원으로 **95% 문제를 원격 해결**
- 참고: wartsila.com/wartsila-online

### 콩스버그 (Kongsberg) — K-IMS + KM Performance (웹 기반)
- **앱보다 웹 중심** (K-IMS는 웹 브라우저 기반 정보 관리 시스템)
- 선박-육상 간 데이터 실시간 동기화, 서비스 요청, 문서/이력 관리
- 2026년 2월 KM Performance 런칭 — 기존 K-IMS, Vessel Insight, K-Fleet 등을 하나의 웹 생태계로 통합
- Remote Diagnostics, Remote Virtual Services 등 원격 서비스 운영
- 참고: kongsberg.com/maritime/products/digital/

### MAN + Rolls-Royce — mýa
- 경쟁 OEM끼리 오픈 플랫폼 구축 → 선주에게 "하나의 창구" 제공

### 3사 공통점
1. 웹 기반 (브라우저 접속)
2. 고객 셀프서비스 (서비스 요청, 이력 조회, 문서 열람)
3. 이력 관리
4. 데이터 시각화 (대시보드/그래프)
5. 플랫폼 통합

## 우리 회사 현황

### 강점
- AS 네트워크 9,890척 (2024 3Q 누적)
- Hi-4S (24/7 스마트케어), ISS 2.0, OceanWise (AI 항로최적화)
- 디지털 관제센터 350척+ 모니터링
- HD현대 건조 선박의 유일한 AS 사업자 (독점)
- 엔지니어 무상 헬스체크 → 리포트 → 수리 권고 → 수주 (검증된 비즈니스 모델)

### Gap (부족한 것)
- 고객용 웹 포털 없음 (전화/이메일)
- 서비스 요청 온라인화 없음
- 호선정보 고객 조회 불가 (내부 시스템)
- 서비스 이력 고객 조회 불가
- 기술 문서 온라인 접근 불가 (요청 시 제공)
- 리포트 시각화 없음 (텍스트/숫자 나열)

## 제안 — 4대 방안

### 방안 1. 고객용 웹 포털 구축 (핵심)

**9개 기능 영역 설계 완료:**

| 영역 | 핵심 기능 |
|------|----------|
| A. 대시보드 | 내 선박 목록 + 상태, 진행 중 서비스, 알림, 빠른 메뉴 |
| B. 호선정보 | 선박 검색, 배전반 스펙/모델/설치연도, 회로도, 기자재 목록, 설치/교체 이력 |
| C. 서비스 요청 | 7단계 폼 (선박→유형→설비→내용→첨부→일정→긴급도), 자동 PM 배정 |
| D. 서비스 추적 | 6단계 스텝바 (접수→PM배정→T/S→견적→공사→완료), 실시간 추적, 메시지 소통 |
| E. 점검/수리 이력 | 헬스체크 이력, 수리 이력, 타임라인 뷰 |
| F. 리포트 대시보드 | 건강도 게이지, 절연저항 추이 그래프, 차단기 막대차트, 부품 수명 프로그레스바, AI 분석 코멘트 |
| G. 기술 문서 | 매뉴얼, 단선도, 부품 카탈로그, 기술회보, 시험성적서, 인증서 (호선별 자동 매칭) |
| H. 알림 | 점검 시기, 부품 교체, 입거, 서비스 진행, 기술회보, 리포트, 미응답 (7종) |
| I. PM 백오피스 | AI 메일 분석, 건별 카드, 인수인계, 견적 매칭, 수기발주서 알림 |

**UI 목업 8개 화면 HTML로 제작 완료** (웹포털_UI_목업.html)

### 방안 2. 라이프사이클 계약 (Lifecycle Agreement) 도입

**현재**: 건별 일회성 수익 (고장 나야 매출)
**제안**: 바르질라 벤치마킹 → 3티어 연간 계약

| Tier | 내용 | 계약 기간 |
|------|------|----------|
| Basic | 연간 헬스체크 + 시각화 리포트 + 웹 포털 + 알림 | 1~2년 |
| Standard | + 기술 지원 무제한 + 우선 대응 + 전담 PM + 부품 수명 관리 | 2~3년 |
| Premium | + 수리 비용 포함(정액) + 긴급 SLA + 원격 지원 + 정비 계획 수립 + 교육 | 3~5년 |

**이득**: 반복 매출(₩10억+/년 가정), 고객 lock-in, 마진 개선, 무상 헬스체크 유상화
**도입 순서**: 파일럿 2~3곳 → Tier 1 계약 → 성과 확인 → 확대 + 업그레이드

---

### 방안 3. 원격 기술 지원 (Remote Expert Support)

**현재**: 전화 → 출장 (수일~수주, 해외 1~2주)
**제안**: 3단계 도입

| Level | 방식 | 도입 시점 | 비용 |
|-------|------|----------|------|
| Level 1 | 전화/영상통화 (기존 스마트폰) | 즉시 | 0원 |
| Level 2 | 화상 플랫폼 (화면 공유, 녹화, 포인터) | 3~6개월 | 낮음 |
| Level 3 | AR 원격 가이드 (스마트글래스) | 12개월~ | 중간 |

**바르질라 실적**: 95% 원격 해결. **미 해군**: AR 지원 시 수리 47% 빨라짐.
**이득**: 출장비 50% 절감(연 ₩1.5억), 대응 속도(수주→수시간), 엔지니어 생산성 3~5배

---

### 방안 4. PM 업무 AI 자동화

**현재 PM 업무 문제:**
- 모든 업무가 메일 기반 → 정보 분산
- 반복 작업이 70% (메일 파악, 과거 건 검색, 견적 산출)
- 인수인계가 메일 50통 포워딩 → 1~2주
- 긴급공사 수기발주서 누락 → 하도급법 위반 리스크

**AI 해결:**
- 메일 AI 분석: HMS 메일 → 3초 만에 선박/일정/Work Scope/견적 자동 추출
- 건별 카드 자동 생성 → 인수인계 30분
- 과거 유사 건 매칭 + 협력업체 추천 + 러프 견적 자동 산출
- 수기발주서: 긴급공사 + PO 미확인 시 자동 드래프트 + 알림

---

## 리포트 고도화 (별도 제안)

**현재**: 텍스트/숫자 나열
**제안**: 시각화 + AI 분석

| 요소 | 시각화 |
|------|--------|
| 종합 건강도 | 게이지 차트 (100점) |
| 절연저항 | 꺾은선 그래프 (12개월 추이 + 정상/위험 기준선) |
| 차단기 동작시간 | 가로 막대 (정상 범위 비교) |
| 온도 | 부위별 막대/히트맵 |
| 부품 수명 | 프로그레스 바 + 교체 시기 |
| AI 분석 | "월평균 27MΩ 하락, 6개월 후 위험 진입 예상" 자동 코멘트 |

**핵심**: "같은 데이터도 보여주는 방식이 다르면 선주의 의사결정이 달라진다. 리포트 품질 = 수주 속도."
